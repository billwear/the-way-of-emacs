Getting Started: A Novel Emacs Experience

So where does one start with building a (sci-fi) novel using Emacs and blogging about it at the same time? Novel writing is already a monster accounting problem. There are so many things to track, each creating a thread through the story. Add to that blogging about fiction and toolkit and … makes it much more complex.

Luckily, Emacs has tools that will help, so I'd better start there.

First, I'm going to define four categories of posts:

  1. The Neptune Anomaly: the novel itself, as it evolves. Yes, I'll probably share parts of it, under copyright, of course. We'll tag these posts with :neptune:.
  2. Then there's stuff about how I'm =trying= learning to write fiction, using my own take on traditional novel construction, tagged :fiction:.
  3. Next is the toolkit itself – how I use the features of Emacs and Org-Mode to sidestep the mechanical and memory aspects of crafting a story, leaving my mind free to be creative. Look for the :toolkit: tag for these.
  4. Finally, there are things that don't reliably fit in other categories, things that break the "fourth wall" as it were. Let's tag these as :meta:.

How to tag them? You can tag headlines like this:

* Chapter 1: Orbital Insertion :neptune:

You can also tag entire posts, like this:

#+tags: neptune

I'll probably do both. But how to automatically index them? Well, that's where something called org-ql ("org query language") comes in handy. I'm new to using it, but I have a feeling it will be a big part of this effort. Automating it means writing a little Emacs Lisp. (Look, if you can write a nuanced interaction between two characters, you can write Emacs Lisp – you just have to want to do so.) Here's what that function looks like to start with: 

(defun bill/generate-toolkit-index ()
  "Generate a tag-based index of :toolkit: posts."
  (interactive)
  (let* ((org-dir (expand-file-name "~/usr/projects/way-of-emacs/"))
         (output-file (expand-file-name "the-writers-toolkit.org" org-dir))
         (output-buffer (find-file-noselect output-file)))
    (with-current-buffer output-buffer
      (erase-buffer)
      (insert "#+TITLE: The Writerâ€™s Toolkit\n")
      (insert "#+OPTIONS: toc:nil num:nil\n\n")
      (insert "This is an auto-generated list of posts tagged =toolkit=.\n\n")
      (insert "* Toolkit Posts\n\n")
      (dolist (entry (org-ql-query
                      :select 'element
                      :from org-dir
                      :where '(tags "toolkit")))
        (let* ((buf (marker-buffer entry))
               (file (buffer-file-name buf))
               (heading (org-no-properties (org-element-property :raw-value entry))))
          (insert (format "- [[file:%s][%s]]\n"
                          (file-relative-name file org-dir)
                          heading))))
      (save-buffer)
      (kill-buffer output-buffer)
      (message "Toolkit index updated."))))

You also need to load the Emacs package (".el") called org-ql. You can do this manually or automatically, but since I write on any device I can get my hands on, wherever I am, I need Emacs to take care of this, so I employ use-package:

(use-package org-ql)

We can talk later about how my init files are structured into different types of starting operations in .emacs.d/lisp/.
